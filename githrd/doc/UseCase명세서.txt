제목 :
	로그인
	
개요 :
	본 유즈케이스는 시스템에 진입하기 위해서 로그인을 제공하는 기능이다.

관련 액터 :
	고객(일반사용자), 관리자

기본흐름 :
	1. 고객은 시스템에 로그인 화면을 요청한다.
	2. 고객은 아이디와 비밀번호를 입력한다.
	3. 고객은 로그인 버튼 누른다
	4. 시스템은 고객의 정보를 확인한다.
	5. 고객 정보가 일치하면 메인 화면으로 이동시킨다.

대체흐름 :
	1-A. 회원만 사용할 수 있는 기능을 선택한 경우 자동으로 로그인 화면으로 이동
	2-A. 아이디와 비밀번호는 해당 규칙을 지켜야 하며 규칙에 위배된 내용이 입력되면 해당 입력란을 초기화
	5-A. DB에 회원 정보가 없으면 다시 로그인 화면으로 이동시킨다.

예외흐름 :
	5-A. DB 접속이 원활하지 않을 경우 에러 페이지로 이동

확장점 :
	1. 고객이 아닌 경우 회원가입 버튼 제공
	2. 회원 가입 시 자동 로그인 된 것으로 처리
	3. 아이디 분실 고객을 위해 아이디 찾기 기능 버튼 제공
	4. 비밀번호 분실 고객 위해 비밀번호 찾기 기능 버튼 제공

제목 :
	회원가입
	
개요 :
	본 유즈케이스는 로그인을 위해 회원가입을 제공하는 기능이다.

관련 액터 :
	고객(비회원)

기본흐름 :
	1. 고객은 시스템에 회원가입 화면을 요청한다.
	2. 고객은 회원이름, 아이디, 비밀번호, 비번체크, 회원메일, 전화번호, 성별, 아바타를 입력한다.
	3. 아이디 입력 시 중복 입력 체크를 한다.
	4. 비밀번호는 두 번 입력하여 두 내용을 비교하여 같은 경우에만 통과시킨다.
	5. 성별 선택 시 해당 성별의 아바타만 노출한다.
	6. 아바타 중 하나만 선택한다.
	7. 고객은 회원가입 버튼 누른다
	8. 회원가입 완료 시 자동 로그인 처리 후 메인 페이지로 이동시킨다.

대체흐름 :
	1-A. 회원만 사용할 수 있는 기능을 선택한 경우 자동으로 로그인 화면으로 이동
	2-A. 이름(한글 4글자), 아이디(영문숫자 포함 8글자), 비밀번호(12345), 메일(도메인 제외 영문 7글자), 전화번호(xxx-xxxx-xxxx)는 해당 규칙을 지켜야 하며 
		 규칙에 위배된 내용이 입력되면 해당 입력란을 초기화
	3-A. 아이디 중복 시에도 해당 입력란 초기화.
	4-A. 두 비밀번호가 일치하지 않을 경우 입력란 초기화.
	8-A. 회원가입 실패 시 다시 회원가입 페이지로 이동

예외흐름 :
	8-A. DB 접속이 원활하지 않을 경우 에러 페이지로 이동
